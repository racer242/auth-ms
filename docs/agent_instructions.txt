При добавлении нового функционала:
- проверяй, нужны ли настраиваемые параметры и предусматривай их в .env.
- Добавляй юнит-тесты.

Для каждой итерации разработки:
- компиляция должна заканчиваться отсутствием ошибок и предупреждений в консоли. 
- проверяй прохождение юнит-тестов. При наличии ошибок - исправляй их.
- обновляй в readme.md описание функционала.
- заноси в changelog.md краткое описание нового функционала и изменений.
- сопровождай коммитом в Git. Коммиты Git необходимо называть по-английски.
